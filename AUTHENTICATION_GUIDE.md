# 🔐 Страницы Логина и Регистрации - Готовы к использованию!

## Как пользователю зарегистрироваться или войти?

### 1. **Регистрация (Создание нового аккаунта)**

Нажмите кнопку **"Зарегистрироваться"** на странице входа:

**Страница регистрации содержит:**
- Поле для логина (минимум 3 символа)
- Поле для пароля (минимум 6 символов)
- Поле для подтверждения пароля
- Кнопка "Зарегистрироваться"
- Ссылка для возврата на вход

**Что происходит после регистрации:**
- ✓ Новый аккаунт создается с начальным балансом 1000 ₴
- ✓ Статус устанавливается как "pending" (в ожидании)
- ✓ Пользователь автоматически входит в приложение
- ✓ Данные сохраняются в localStorage

### 2. **Вход (Логин в существующий аккаунт)**

**Страница входа содержит:**
- Поле для логина
- Поле для пароля
- Кнопка "Войти"
- Ссылка на регистрацию
- **Тестовые креденшалы для демонстрации:**
  ```
  Логин: testuser
  Пароль: 123456
  ```

**Что происходит после входа:**
- ✓ Данные пользователя загружаются
- ✓ Токен сохраняется в localStorage
- ✓ Отображается основное приложение
- ✓ В меню "Поще" видна информация профиля

### 3. **Вход из основного приложения**

**В меню "Поще" (More):**
- 👤 **Профиль** - показывает логин и статус пользователя
- 💬 **Подтримка** - чат с поддержкой
- ⚙️ **Налаштування** - настройки
- 📋 **Документи та умови** - документы
- 🚪 **Вийти** - кнопка для выхода из аккаунта

**Кнопка "Вийти" (LOGOUT):**
- Удаляет данные пользователя из localStorage
- Удаляет токен авторизации
- Возвращает пользователя на страницу входа

## 🔧 Практическая информация

### Тестовые аккаунты:
```
1. testuser / 123456 (баланс: 125738 ₴, статус: active)
2. testadmin / ByhFt{GYcPX?#zB|okk~ (баланс: 50000 ₴, статус: active)
3. testuser5 / pass123 (баланс: 10000 ₴, статус: active)
```

### Backend API endpoints:
- `POST /users/login` - вход в аккаунт
- `POST /users/register` - создание нового аккаунта
- `GET /users/me` - получить информацию текущего пользователя

### Хранилище данных:
- Данные пользователя хранятся в `/server/db.json`
- Сессия пользователя хранится в `localStorage` приложения
- Токен авторизации сохраняется как `authToken`

## 📝 Взаимодействие с App.js

Приложение автоматически:
1. Проверяет наличие токена при загрузке
2. Показывает страницу входа, если пользователь не авторизован
3. Загружает основное приложение, если авторизация успешна
4. Передает информацию пользователя в компонент MorePage
