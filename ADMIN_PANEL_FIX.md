# Исправление админ-панели и страницы проверки паспорта

## Проблема
При регистрации новых пользователей отображалась страница "Проверка паспорта", но админ-панель не открывалась - она загружалась на странице входа вместо админ-интерфейса.

## Причина проблемы
1. **URL содержал символ `#` в пути**: `/ByhFt{GYcPX?#zB|okk~` интерпретировался браузером как:
   - pathname: `/ByhFt{GYcPX?`
   - hash: `#zB|okk~`
   
2. **Проверка админки была неправильной**: Код проверял только `window.location.pathname`, что не совпадало с реальным путем.

3. **KYC статус блокировал админку**: Когда админ открывал админку, в localStorage могли быть данные обычного пользователя, что устанавливало KYC статус как 'pending' и блокировало админку.

## Решение

### 1. Создана функция проверки админки
```javascript
const isAdminPath = () => {
  return (
    window.location.hash.includes('ByhFt{GYcPX?') ||
    window.location.pathname.includes('admin') ||
    window.location.pathname === '/ByhFt{GYcPX' ||
    new URLSearchParams(window.location.search).get('admin') === 'true'
  );
};
```

Теперь админка доступна через несколько способов:
- `/ByhFt{GYcPX?#zB|okk~` (исходный способ, теперь работает)
- `/admin` (новый удобный способ)
- `/?admin=true` (способ через query параметр)
- Любой URL с хешем `#ByhFt{GYcPX?` (часть пароля)

### 2. Админка не загружает user данные
В `useEffect` добавлена проверка:
```javascript
if (isAdminPath()) {
  return; // Не загружаем user данные для админки
}
```

Это предотвращает установку KYC статуса и блокировку админки.

### 3. Добавлена проверка admin первой
Проверка админки выполняется ДО всех остальных проверок авторизации, поэтому админка открывается независимо от состояния авторизации.

### 4. Добавлена обработка состояния 'loading'
Когда пользователь регистрируется, KYC статус загружается асинхронно. Теперь во время загрузки показывается экран с сообщением "Загрузка...".

## Поток работы теперь корректный

### Для новых пользователей:
1. Пользователь регистрируется → `status: 'pending'`
2. Показывается страница "Проверка паспорта" → пользователь ждет одобрения
3. Админ одобряет в админ-панели → `status: 'approved'`
4. Пользователь видит основное приложение

### Для администратора:
1. Админ открывает админ-панель (любым удобным способом)
2. Админ видит все разделы: KYC, Пополнения, Переводы, Пользователи, Чат
3. Админ может одобрить/отклонить запросы пользователей

## Способы открытия админ-панели

**Рекомендуемые способы:**
- `http://localhost:3000/?admin=true` ✅ (самый удобный)
- `http://localhost:3000/admin` ✅ (понятный)
- `http://localhost:3000/ByhFt{GYcPX?#zB|okk~` ✅ (исходный, но менее удобный)
