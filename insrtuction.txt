# ПРОЕКТ: NeoBank MVP (с ручным управлением через админа)

## 🎯 Цель

Создать веб-приложение в стиле нео-банка (как Monobank), но с упрощённой логикой:

* Без реальных банковских интеграций
* Все операции обрабатываются вручную через админку
* Основной упор на UI и простую логику

---

## 🧠 Основная идея

Это НЕ настоящий банк.

Это:

* красивый интерфейс как у банка
* простой backend
* все действия контролируются администратором

---

## 🧩 Архитектура

### Frontend

* React
* Простая структура
* Компонентный подход
* Mobile-first дизайн

### Backend

* Node.js (Express)
* REST API
* Без усложнений

### База данных

* PostgreSQL (или простая альтернатива)
* Минимум связей

---

## ⚙️ Основные правила логики

1. НЕТ реальных платежей
2. НЕТ внешних API
3. ВСЁ проходит через админа
4. ВСЁ работает через статусы

---

## 👤 Статусы пользователя

* pending — ждёт проверки
* approved — активен
* rejected — заблокирован навсегда

---

## 🪪 KYC

Пользователь загружает:

* паспорт (лицевая сторона)
* паспорт (обратная сторона)

Админ:

* одобряет или отклоняет

Повторная загрузка после отказа НЕ допускается

---

## 💸 Пополнение

Пользователь:

* выбирает способ
* вводит сумму

Система:

* создаёт заявку (pending)

Админ:

* вручную добавляет реквизиты
* подтверждает
* вручную увеличивает баланс

---

## 💳 Баланс

* хранится как число
* изменяется только админом

---

## 🔁 Переводы и платежи

Это НЕ реальные переводы.

Это:

* заявки
* обрабатываются вручную админом

---

## 🧾 История операций

Простая таблица:

* тип
* сумма
* комментарий

---

## 🛠 Админка

Админ может:

* одобрять KYC
* управлять пользователями
* обрабатывать заявки
* менять баланс
* создавать операции вручную

---

## 🎨 UI

* стиль как у Monobank
* минимализм
* мобильный интерфейс

---
